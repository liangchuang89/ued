      IData¶

      提供一种数据注入的方式，以baobei-w页面为例： 系统会在搜索templates下的baobei-w时，搜索相同目录下是否存在data.php，并判断其中是否包含一个实现了IData接口的类，类名必须为页面名+data,无-
      比如: 

      <?php
class baobeiwdata implements IData
{
  function getData($key)
  {
      switch($key){
          case 'personal-custom':
              $data = iconv('GBK','UTF-8','{"customed":"custom-btn-customed"}');
              break;
          default:
              $data = null;
      }
      return $data;
  }
}
?>
      这样就会将personal-custom组件的原来的{"customed":""}和{"customed":"custom-btn-customed"}混合，类似KISSY.merge()。

     )

      • use:插件都使用了那些参数
      • description:插件作用描述

      P¶

      新的使用$_GET参数的方式

      原来在页面里面我们的使用GET的方法： 
      <?php
if(isset($_GET['get'] && $_GET['key'] == 'value')
?>
      这种方式有两个缺点：
      • 随着系统的增长，系统中不明所以的$_GET参数会越来越多
      • 很烦不好打

      template_inc里提供这样的方式：

      <?php
//extend为$_GET中的键值，第二个参数为注释会在log工具中展现，第三个为了兼容编码，默认$gbk为true
P::recordGET('extend','添加Extend功能', false);//记录当前参数功能
P::get('extend');//使用这个参数
P::set('extend','off');//设置这个参数默认值
?>
      recordGET之后，会将注释记录到L（日志）中,详见LogRecordCommand.php；

      TI_Autoloader¶

      自动加载类 例子： 

      //无参数，则默认根路径为autoload.php文件所在目录的绝对路径
$ti_autoloader = new TI_Autoloader();
//也可以手动设置一个根路径
//$ti_autoloader = new TI_Autoloader('/user/dd');
$ti_autoloader->add(array(
 "default"=>array("core","event","common"),//默认的包，会加载这些包下的所有文件
 "auto"=>array("base")//autoload机制要遍历的包，这个包下的类实例化的时候会自动加载
));
